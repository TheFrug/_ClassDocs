Today's Work Objective:

Behavior System - Project Hikikomori

At any given time, Hiki is doing at most one type of "Behavior". 
While doing behaviors, Hiki's resource bars, Stress, Spoons, Hunger, Cash, and Time will change values over time. 

Behaviors can either be default, toggled, or activated (a better term might be used here):

- Default behaviors are idle tasks like doomscrolling, napping, using the computer, etc. Hiki will default to one of these when the player does not input any actions for Hiki to take.
As Hiki spends time doing these tasks, resources will change by steady and sproadic amounts. For example, Doomscrolling eventually causes spikes in stress which will quickly accumulate if the player isn't careful.

- Activated behaviors each take a fixed amount of time, during which Hiki's current behavior will be set to the activated one. Over this fixed time, Hiki's resources will change.
The rate of change should be calculated based off the change (like delta) over the time amount for that behavior.
