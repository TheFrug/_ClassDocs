Random Thoughts and Internal Dialogue

Currently, my unity project has a system where lines of dialogue are presented via a ThoughtBubble object which appears as a small thought bubble with text and potentially a name plate. This box is meant to float upwards and eventually fade out, with travel speed and fadeout time set by me. These can be spawned a few ways, but the goal is for them to be spawned purely via a method (currently called SpawnThought, will change to InitThought or similar) that passes in a Thought object, which is a scriptable object that contains the SpeakerKey parameter, a type of either automatic or interactive, a lifetime, a rise speed, and a link to a specific yarn script. When the InitThought method is run, the Thought scriptable object passes in its info into the ThoughtBubbleView object in my scene, which is also a yarnspinner DialoguePresenterBase, and this one also handles option presentation. Once the method starts, each line in that yarnspinner script will be converted into a ThoughtBubble instance with text content, color, and speaker name all dictated by a combination of Thought's data as well as the information from FamilyManager, which is read via SpeakerKey.


The real change we want is to make it so that thoughtBubbles don't just slide up past the canvas mask on the ThoughtBubblePanel, but rather they float to the top like balloons, only fading away after their lifetime has ended. If multiple thoughts are on screen at once, I want them to stack top to bottom like balloons floating to a decreasingly lower ceiling, with each ThoughtBubble's floating stopping once their top edge hits the bottom edge of another Thought OR the upper edge of the THoughtBubblePanel object. Then, they stay there until their lifetime ends, then they float away and slowly fade out.
